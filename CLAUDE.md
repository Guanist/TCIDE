# AI 编程统一规范
适用环境：TCIDE / Trae / QClaw
核心目标：保障代码质量 + 严格控制 Token 消耗，规则优先级最高

## 第一部分：Karpathy 编程准则（编码行为约束）
### 1. 先澄清，再实现
编写代码前复述任务目标、交付结果与边界约束，确认无误再执行。
存在模糊点仅提出1-2个精准问题，禁止自行猜测、默认补充需求。
标准确认话术：我将使用指定方案实现对应功能，严格遵循约束条件，是否确认？

### 2. 极简实现，拒绝过度设计
仅解决当下明确需求，不提前做抽象、预留扩展、额外功能镀金。
优先使用项目现有代码风格与逻辑，代码以简短、易读、够用为第一原则。
杜绝炫技写法、冗余架构，能用少量代码完成则绝不扩容篇幅。

### 3. 微创修改，只改必要代码
仅改动当前任务直接关联的代码区块，不执行无关重构、代码格式化、顺手优化。
对未完全理解的原有代码、注释保持原样，不擅自调整。
仅清理本次新增产生的无效代码，项目原有废弃代码只标记、不删除。

### 4. 增量开发，闭环验证
采用「实现-自测-验证」的增量流程推进开发。
代码修改完成后主动校验核心功能，明确反馈运行状态或现存问题。
遇到阻塞、无法完成的场景必须显性标注，不静默终止流程。

## 第二部分：上下文与 Token 管控规则（核心节流）
### 1. 文件加载限制
禁止自动遍历、读取整个项目文件树。
单次上下文仅加载：当前编辑文件 + 最多 3 个直接依赖文件。
如需查阅更多文件，必须等待用户明确指定路径/文件名。

### 2. 会话历史管理
仅保留**最近3轮对话记录**，更早交互内容自动截断，不纳入上下文。
历史对话仅用于辅助理解当前任务，不得重复复述过往代码与结论。

### 3. 长期静态记忆
项目架构、技术栈、全局配置、接口说明统一存放于 `.tcide/context/static.md`。
该部分作为静态缓存内容，单次加载后复用，不重复全量传输、不产生冗余 Token。

### 4. 输出内容规范
仅输出新增、修改、修复的代码片段，不完整重复已有文件代码。
注释精简务实，不添加无意义描述；回答直击要点，减少铺垫性文字。

## 补充约束
本文件所有规则拥有最高执行优先级，全程严格遵守，不得主动规避。
