# 虎猫 TCIDE - 技术规格说明书

## 1. 项目概述

- **名称**: 虎猫 TCIDE
- **类型**: Electron 桌面客户端（Windows 专用）
- **定位**: 纯本地、无限制、零冗余的个人专属 AI 编程 IDE
- **核心特性**: 双智能体架构（Builder + Coder）、全自动化工程闭环、工程文件互操作

## 2. UI/UX 规格

### 2.1 布局结构
- **三栏固定布局**（不可变）:
  - 左侧: 文件树面板（宽度 240px，可拖拽调整）
  - 中间: 代码编辑器（自适应剩余宽度）
  - 右侧: AI 对话 + 任务执行面板（宽度 360px，可折叠）
- **底部状态栏**: 高 24px，显示 Git 分支、编码、光标位置
- **顶部标题栏**: 使用系统原生窗口框架（无自定义标题栏）

### 2.2 视觉风格
- **暗色主题**（唯一默认主题）:
  - 主背景: `#1E1E1E`
  - 面板背景: `#252526`
  - AI 面板背景: `#2D2D30`
  - 主要文字: `#D4D4D4`
  - 次要文字: `#858585`
  - 强调色: `#007ACC`
  - 边框色: `#3C3C3C`
  - 悬停色: `#2A2D2E`
  - 选中色: `#094771`
- **字体**:
  - 编辑器: `JetBrains Mono`, `Cascadia Code`, `Consolas`, monospace
  - UI: `Segoe UI`, system-ui
  - 行高: 1.6，字间距标准
- **动画**: 零动画，零过渡效果

### 2.3 组件清单

| 组件 | 功能 | 状态 |
|------|------|------|
| 文件树 | 显示项目文件结构，支持新建/删除/重命名/拖拽 | 必需 |
| 编辑器 | Monaco Editor，支持语法高亮/折叠/多光标/查找替换 | 必需 |
| AI 面板 | 流式 Markdown 输出，对话历史，终止任务按钮 | 必需 |
| 任务面板 | Builder/Coder 任务状态展示，进度条 | 必需 |
| 终端 | xterm.js，可显示 build 输出 | 必需 |
| 状态栏 | Git 分支、编码、行号、光标列 | 必需 |
| 设置抽屉 | 模型选择/API 配置/快捷键配置 | 必需 |
| Zen Mode | 一键隐藏左右面板，编辑器全屏 | 可选 |

## 3. 功能规格

### 3.1 双智能体架构
- **Builder Agent**: 接收需求 → 架构分析 → JSON 任务拆分
- **Coder Agent**: 接收任务 → 文件读写 → 终端执行 → 结果反馈
- **通信机制**: SQLite 共享"项目记忆"表
- **调度器**: 本地任务队列，支持断点续做

### 3.2 项目索引引擎
- 启动时解析项目结构，生成四层上下文索引存入 SQLite:
  1. 文件树摘要
  2. 模块摘要
  3. 符号图谱（类/函数/依赖关系）
  4. 具体代码（按需加载）
- 支持 Android Gradle 项目识别

### 3.3 自动化任务闭环 (TaskRunner)
- 流程: 拿任务 → Coder 生成代码 → 编译验证 → 错误修复 → 重试（最多3次）
- 编译命令自动检测（Gradle/Maven/npm/其他）
- 通过 diff 摘要生成 Conventional Commits

### 3.4 模型适配层
- 支持: DeepSeek-V3, DeepSeek-R1, DeepSeek-Coder, Ollama, 自定义 OpenAI 兼容接口
- 配置加密存储（electron-store + safeStorage）
- Builder 默认 R1，Coder 默认 V3/Coder

### 3.5 工程文件互操作
- 兼容 `.qclaw/`、`.codebuddy/`、`.trae/` 三种工程配置
- 打开项目时自动识别并转换
- 保存时同步写入所有已启用的配置文件夹

### 3.6 网络与隐私
- 全局请求拦截器：默认阻止非 localhost/用户配置端点的请求
- 无 DNS 预解析，无 WebRTC，无第三方 CDN
- 字体和静态资源内嵌

## 4. 技术栈

| 层次 | 技术 |
|------|------|
| 框架 | Electron 33 |
| 编辑器 | Monaco Editor（裁剪后） |
| 终端 | xterm.js |
| 数据 | better-sqlite3（项目记忆）、electron-store（配置） |
| 构建 | Vite + TypeScript + electron-builder |
| AI | DeepSeek API / Ollama / 自定义兼容接口 |

## 5. 性能目标

| 指标 | 目标 |
|------|------|
| 安装包体积 | ≤ 80MB |
| 启动内存 | ≤ 180MB |
| 启动时间 | ≤ 2 秒 |
| 长期稳定性 | 24h 内存增长 ≤ 50MB |

## 6. 验收标准

1. 三栏布局正确渲染，响应流畅
2. Builder 能正确拆分 Android/Kotlin 需求为 JSON 任务列表
3. Coder 能正确执行文件读写和终端命令
4. TaskRunner 编译-错误-修复-重试闭环正常运作
5. DeepSeek API 流式输出正常
6. 工程文件兼容适配器正确读写三套配置
7. 无任何非授权网络请求（Wireshark 验证）
8. 打包后 EXE 可独立运行
