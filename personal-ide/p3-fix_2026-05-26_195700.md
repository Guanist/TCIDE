# P3: Provider 配置驱动 — 2026-05-26 19:57

## Summary
Removed all provider-specific hardcoding in Settings UI. Model list and provider configuration are now fully data-driven via `modelRegistry`.

## Changes

### index.html
- **Removed** hardcoded model items (DeepSeek-V3, DeepSeek-R1)
- **Removed** 3 separate provider config sections (`deepseek-config`, `huoshan-config`, `custom-config`)
- **Added** single generic `#cfg-base-url` input (no per-provider variants)
- **Added** `#cfg-api-protocol` select (auto-populated by provider)
- **Added** `#cfg-meta-display` read-only metadata panel (ctx/max tokens/cost/reasoning)
- **Added** Anthropic to provider dropdown

### main.ts (refactored 6 functions)
| Old | New |
|-----|-----|
| `BUILTIN_MODELS` const (hardcoded) | `loadModelList()` + `renderModelList()` (dynamic from registry) |
| `updateSettingsUI()` (3 base URL inputs) | Single `#cfg-base-url` update + `updateProviderDefaults()` + `updateModelMetaDisplay()` |
| `updateProviderConfig()` (per-provider switch) | `updateProviderDefaults()` (generic: auto-fill baseUrl + protocol + docs links) |
| `selectBuiltinModel()` (hardcoded defaults) | Uses `modelListCache` data for defaults, auto-selects builder model if reasoning-capable |
| `getEffectiveBaseUrl()` (4-case switch) | Single `#cfg-base-url` read |
| 5 static model-item click bindings | Dynamic binding in `renderModelList()` from API data |

### New functions
- `loadModelList()` — calls `window.api.listModelMeta()`, caches to `modelListCache`
- `renderModelList()` — groups by provider, renders dynamic HTML, binds click events
- `updateProviderDefaults()` — auto-fills baseUrl, protocol, docs links on provider change
- `updateModelMetaDisplay()` — queries `getModelMeta()` and shows ctx/maxTokens/cost/reasoning

### main.css
- New styles: `.model-item`, `.provider-section`, `.model-meta-display`, `.meta-item`, `#cfg-api-protocol`
