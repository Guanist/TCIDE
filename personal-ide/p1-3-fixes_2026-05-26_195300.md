# P1-3: Model Metadata Registry — 2026-05-26 19:53

## Summary
Added model metadata registry with 22 built-in models across 5 providers.
Adapter now uses per-model cost instead of hardcoded Doubao-lite prices.
Both main and renderer compile with zero errors.

## New File
`src/core/model/model-meta.ts`

### ModelMeta interface:
```ts
{ id, provider, name, contextWindow, maxTokens, reasoning,
  cost: { input, output, cacheRead?, cacheWrite? },
  capabilities: string[], supportsStreaming }
```

### Built-in models (22):
| Provider | Models |

| huoshan | doubao-lite-32k, doubao-pro-32k, doubao-pro-128k, doubao-1.5-pro-256k, doubao-1.5-thinking-pro, deepseek-v4-pro |

| custom | gpt-4o, gpt-4o-mini, o3-mini, qwen-plus, qwen-max |

### ModelRegistry methods:
- `lookup(provider, modelId)` — exact match
- `lookupById(modelId)` — cross-provider match
- `listByProvider(provider)` — filter
- `listAll()`
- `register(meta)` — add/override custom models
- `getCost(provider, modelId)` → `{input, output} | null`
- `getContextWindow(provider, modelId)` → `number | null`
- `getMaxTokens(provider, modelId)` → `number` (default 4096)
- `supportsReasoning(provider, modelId)` → `boolean`

## Integration
- `adapter.ts`: `recordTokenUsage()` now calls `modelRegistry.getCost()` — cost auto-detected per model, fallback to Doubao-lite default
- `ipc-handlers.ts`: new `model:listMeta` and `model:getMeta` IPC handlers
- `preload.ts`: new `listModelMeta(provider?)` and `getModelMeta(provider, modelId)` API
- `types.d.ts`: new `ModelMeta` interface + TcIdeApi methods
