# P0 Fixes: Token Usage Bug + API Key Encryption — 2026-05-26 19:33

## Summary
Fixed 2 critical P0 bugs found during the TCIDE audit. Both fixes compiled with zero TypeScript errors.

## Fix 1: Token Usage Recording (adapter.ts + ipc-handlers.ts)

**Bug**: `adapter.recordTokenUsage()` used `ipcRenderer.send()` which only works in renderer process. All adapters in ipc-handlers.ts run in main process → usage data silently dropped.

**Fix**:
- `adapter.ts`: Added `onUsage` callback property to ModelAdapter class
- `adapter.ts`: `recordTokenUsage()` now checks `this.onUsage` first, falls back to `ipcRenderer.send()`
- `ipc-handlers.ts`: New `createAdapterWithUsage()` factory that injects `onUsage` callback calling `insertUsage()` directly
- `ipc-handlers.ts`: New `insertUsageRecord()` shared helper (used by both `record-usage` listener and `createAdapterWithUsage`)
- Replaced 6 of 7 `new ModelAdapter(config)` → `createAdapterWithUsage(config)` (testConnection kept as-is)
- Added `ipcMain.on('record-usage', ...)` for renderer-side adapter fallback

## Fix 2: API Key Encryption at Rest

**Bug**: `model:saveConfig` stored apiKey as plaintext in electron-store JSON file.

**Fix**:
- `model:saveConfig`: Encrypts apiKey with `safeStorage.encryptString()` before store
- `getModelConfig()`: Decrypts apiKey with `safeStorage.decryptString()` on read
- Compatible with legacy plaintext keys (try-catch silently keeps unencrypted values)
- Graceful when safeStorage unavailable (keeps plaintext)
